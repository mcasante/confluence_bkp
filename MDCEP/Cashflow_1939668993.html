Marcado de Capitais : Cashflow

Created by Guilherme Pettene Dantas, last modified on Jun 20, 2022

Resumo executivo:
Métricas da carteira do nosso portfólio são calculadas a partir da cashflow, entretanto, existem inconsistências nos dados que precisam ser corrigidos.

Contexto e Problema que queremos resolver:
Métricas de inteligência da carteira como PDD, inadimplência, maior devedor do portfólio etc. são todas calculadas a partir da cashflow. Além disso, a base serve como source of truth para relatórios para investidores. Portanto, é necessário validar os campos e adicionar variáveis.

O que já foi descoberto:

Falhas:

  • Expected date: substituição das datas de vencimento. Cliente pede novo boleto, a tabela corrige a data antiga pela data do novo boleto

  • Lidar com pagamentos parciais, inserção de colunas com os residuais de principal e juros 

  • coluna com o valor presente dos valores esperados

  • Contrato renegociados não são rastreáveis

  • formatação das colunas, taxas e datas vem com formatos difíceis de serem manipulados

  • Não ter PDD

  • Não sabemos o que a cashflow caracteriza como upsell

  • Todos os meses o time de alocação faz o download da cashflow, e alguns contratos desaparecem com o tempo. → Para alguns casos, os contratos possuem seus números alterados por isso desaparecem.

  • Pagamentos realizados estão sendo salvos no banco de dados com uma data futura

    A função que está registrando os pagamentos da parcela no sistema está colocando a data de vencimento como data de pagamento, já coloquei aqui no backlog para fazer um fix desse script e tbm ajustar a base, pq nós temos a data certa de pagamento só está em outra coluna. A data certa tá na tabela “asset.realized_cashflow_entry” a “coluna credit_date”



O que não sabemos:

Como é configurado o upsell na cashflow?

Mudanças dos usuários no midgard alteram os dados nobanco, consequentemente, alterando a cashflow?
