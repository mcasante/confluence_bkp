Project: CX : Transbordo de demandas de cobrança

Created by Stela Schenato, last modified on Jun 21, 2022

Início das conversas:

Reunião de alinhamento 17/06/22.

Presentes: Aline Paiva, Cristopher David, Giovana Anielle, Patricia Azeredo, Thiago Silvestre, Nathália Cardoso e Stela Schenato.

A conversa foi iniciada como iniciativa do time de customer para debater sobre processo de atendimento de demandas de cobrança.

O direcionamento de médio prazo é acompanhar a integração da Cobmais com a Zenvia e Hubspot.

A Patrícia informou que foi pago para a Cobmais a estruturação de uma integração da ferramenta com a Zenvia, dessa forma a possível transição para a Intercom não estaria alinhado com o planejamento de cobrança, pois a Patrícia não foi incluída nas conversas.

Além disso, estão conversando com a Cobmais sobre uma possível integração com o hubspot.

Dessa forma, devemos acompanhar juntos essas integrações para que possamos formatar um modelo de transbordo para o suporte nível 2 que será realizado pelo time de cobrança.

O direcionamento de curto prazo é uma nova maneira de encaminhar as demandas para o time de cobrança por parte do time de atendimento e registro das solicitações no hubspot.

  • Todas as demandas que são direcionados para o time de cobrança devem ser registradas através de ticket no hubspot;

  • As demandas devem ser direcionadas para o time de cobrança por email, por dentro do hubpost e em cópia para o cliente:

  • As demandas devem ser encerradas com a resolution “demanda encaminhada para a área responsável”.

O que precisamos fazer para avançar no curto prazo?

  • Levar as categorias voltadas para cobrança para avaliação do time de cobrança. Segue documento de categorias de tickets: https://docs.google.com/spreadsheets/d/1-ystwHGDf7uoTzkYiMlFACfKpM-Fcrt9Ggi353jDw4E/edit#gid=1642227693

  • Criar templates para auxiliar no transbordo da demanda para cobrança (orientações sobre a demanda e SLA); Documento de SLA para consulta: https://docs.google.com/spreadsheets/d/1QQCq3hFpdDzVTIFS8pN8RxOOUsXix_Y4IVKIqh_pLII/edit

  • Criar dashboard de tickets de cobrança;
