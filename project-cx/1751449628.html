Project: CX : Nova Ativação na experiência de Plataforma v. ago/21

Created by Fernanda Mano, last modified on Aug 11, 2021

Em 09/08/21 alteramos as telas de cadastro da Plataforma para os leads que se cadastram a partir da homepage a55.

Empresas vindas de Campanhas de Marketing, Cadastro via LP de Sales ou Fluxo de Integração não obedecem a esse fluxo até que as LPs sejam alteradas

A nova experiência:

  • colhe mais informações sobre o lead antes que ele chegue as telas de informação de dados financeiros.

  • entrega uma devolutiva de desqualificação nos casos de empresas Sem Oferta de Crédito(Desqualificadas).

  • Elimina a necessidade da etapa [ATV] Dados adicionais cadastrados

OBJETIVO

Lead cadastrar dados financeiros:

Revalidação de informações:

  • Se investe em marketing digital (se sim o montante)

  • Valor solicitado

  • quais bancos/adquirentes trabalha

  • MMR (revalidar)

Lead integrar contas correntes para obter uma primeira proposta

CX correr o processo de checagem de lead qualificado;

Caso lead tenha seguido até aqui sem assistência,

  • Caso qualificado:

    • Checar se dados financeiros foram integrados corretamente e;

    • informar lead sobre volume de receita necessária para seguir para próxima fase (dar assistência caso necessário)

    • informar próximos passos

  • Caso desqualificado: dar no go de acordo com motivo de perda;

PLATAFORMA

Após conta verificada lead entrará em régua de boas vindas e será orientado a cadastrar as informações adicionais/financeiro via Sendgrid. Sem essas informações ele não consegue seguir com o processo.

Após o cadastro das informações adicionais será ativada a régua de comunicação para lead integrar contas correntes na plataforma a55.

Plataforma nada acontece; CX valida se o lead pode seguir em frente ou não.

Uma vez validado, com todas as infos cadastradas, entra em atuação o time de crédito e seu motor. A ideia é que uma proposta esteja disponível para o usuário em poucos minutos e com isso possa seguir as próximas etapas da jornada

Em 11/8: de 2 a 5 dias úteis para resultado de análise

Dados financeiros informados na plataforma.

TAREFAS

O lead será trabalhado via Sendgrid (régua de 5 dias).

Em D+1 da confirmação da conta será criada uma task para o CX Agent entrar em contato para o lead cadastrar dados financeiros.

Cadência aplicada entre 16/8 até 31/08:

  • Cris: 7 follows em 5 dias

  • Stela: 5 follows em 7 dias

  • Após essa data, segue a cadência com melhor conversão

Após a entrada do Deal nesse estágio o CX Agent será notificado e uma task será criada para que o contato seja feito imediatamente. Caso haja um acúmulo de cards nessa etapa o Deal Score deverá ser usado para priorização.

Após a entrada nesse estágio será criada uma task para o CX Agent fazer a verificação de dados financeiros;

Caso precise de contato, CX Agent cria tasks adicionais manualmente para registrar a cadência de contatos

AUTOMAÇÕES

  • Entrada para o estágio com Flag Conta Verificada = true

  • Sorteio do CX Agent responsável.

  • Criar task para entrar em contato em D+1

  • Entrada para o estágio com Flag Cadastro Adicional = true

  • Notificação e criação de task para contato do CX Agent

Entrada para o estágio com Flag CC = true

FINAL

Deal move para o próximo estágio com o cadastro de contas correntes (Flag CC = true)

Deal pode ser movido para Lost: é desqualificado; não tem interesse/segurança em cadastrar contas; não responde a cadência de contatos; indica que não consegue comprovar o volume de MMR com extratos OFX/movimentação bancária; tem MMR efetivo abaixo de 20k.

Se o motor de desqualificação falhar, incluir casos clássicos de empresa desqualificadas

Deal move para o próximo estágio com as contas correntes integradas (Flag CC = true)

Deal pode ser movido para Lost caso não se interesse por cadastrar as informações adicionais ou caso não seja qualificado para uma linha de crédito.

Deal move para outro Pipeline se integra informações o suficiente para ser analisado.

  • Se Prime: Move para [CA] Adicionar documentação pré analise;

  • Se outros: Move para [CA] Empresa pronta para análise;

Deal move para outro estágio [ATV] Lost: se desiste do processo de análise; se não responde a cadência de contatos de CX; indica que não consegue comprovar o volume de MMR com extratos OFX/movimentação bancária; tem MMR efetivo abaixo de 20k.

Se o motor de desqualificação falhar, incluir casos clássicos de empresa desqualificadas

Casos de empresas Sem Oferta de Crédito (Desqualificadas):

  • MEI,

  • <6 meses de vida,

  • Faturamento <20k,

  • Pedida de linha <20k,

  • Setores: Indústria/Fábricas; Comércio físico de produtos;

  • Uso da linha: Repagamento de dívida

*A etapa [ATV] Dados adicionais cadastrados tem se torna obsoleta no momento em que:

(1) LPs de captação de leads estiverem adequadas a nova experiência;

(2) leads da experiência antiga chegarem até o final de fluxo: leads que estiverem em qualquer estágio do pipeline e que não superaram [ATV] Dados adicionais cadastrados até a data de implementação das novas LPs.

Além disso, estamos trabalhando em conjunto com Produto para definir uma data final para dar suporte a experiência antiga.
