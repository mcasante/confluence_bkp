Plataforma a55 : Memo call novo fluxo de ativação da plataforma 04.04

Created by Tech A55 on Apr 04, 2022

Contexto:

Dentro do squad de Alternative Credit, estamos com a iniciativa de reformular toda jornada de contratação de crédito da plataforma, com foco em atacar o step em que temos maior perda de conversão até o aceite de uma proposta firme.

Esse step com maior perda de conversão coincide com o tópico que gera maios tickets de atendimento para o time de CX, que é justamente o fluxo de envio de dados para analise de crédito.

Hoje esse fluxo se resume a etapa de cadastro de contas de adquirentes e de integração de contas bancárias. Sem isso hoje nenhum cliente consegue receber uma proposta firme e seguir com o processo.

O que gostaríamos de testar é uma proposta de passar a gerar propostas firmes (com condições conservadoras) logo no início da jornada, como chamariz para que o cliente envie dados adicionais.

Caso o cliente esteja de acordo com esses termos, nossa proposta é que ele já possa formalizar operação e seguir com a contratação. Caso ele queira termos melhores, a condição é justamente enviar + dados.

Qual o objetivo da reunião?

Validação da v1 do novo fluxo de contratação de crédito “padrão” > foco em leads vindos de forma organica, de campanhas de marketing e de parcerias sem integração

Escopo do fluxo a ser validado:

-Primeiro contato do lead com fluxo de contratação > envio de set de dados inicial para qualificação (go / no-go)

-Geração da primeira proposta firme com dados publicos

-Fluxo de envio da dados adicionais para recebimento de uma proposta personalizada

Premissas:

-Reduziremos perguntas do cadastro inicial, e passaremos a perguntar somente CNPJ, nome completo, e-mail, telefone, cargo + opt-in nos termos e condiçõe da a55

-Passaremos a dar no-go logo após cadastro inicial > dando racional para No-go

-Retornaremos da analise da Vadu já com uma proposta inicial firme, cliente poderá optar por seguir com ela e já formalizar, ou enviar dados adicionais para obtenção de uma proposta personalizada

-Não permitiremos que o cliente altere os termos dessa proposta inicial antes de nos enviar dados adicionais

-Caso em d+1, identifiquemos que cliente possui dados de agenda de recebíveis na CERC, poderemos no dia seguinte a data de inscrição, gerar uma nova proposta melhorada com base nessas informações, sem que o cliente precise enviar dados adicionais

-Propostas evolutivas > seja via CERC ou geradas com base em dados transacionais, não podem nunca ser piores do que a proposta inicial gerada com base na Vadu
