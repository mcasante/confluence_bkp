Parcerias : Iugu - em construção

Created by andreia.brizolla on Aug 04, 2021

O modelo inicial da parceria tem como objetivo validar o potencial de disponibilizar um serviço de crédito dentro da plataforma da Iugu, para tal foi adotado o seguinte modelo:

Onde:

  1. Através de um botão inserido no dashboard do cliente ele solicita o crédito.

    1. Deverá ser implementado um filtro que só possibilite a solicitação de crédito para clientes dentro do ICP da a55 (doc ICP em anexo);

  2. Iugu cria uma lista de solicitações do clientes;

  3. Iugu envia para a55 as solicitações e dados para gerar as propostas. Os dados podem ser enviados em arquivos separados entre dados de cadastro e dados transacionais.

    1. Definir qual será a periodicidade de envio das listas;

    2. Dados de cadastro dos usuário solicitantes (.csv)

      1. cnpj (CNpj da empresa)

      2. email (e-mail do cliente)

      3.  resp_first_name (nome do responsável pelo pedido)

      4.  resp_last_name (sobrenome do responsável pelo pedido)

      5.  fantasy_name (nome fantasia da empresa)

      6.  telephone (telefone contato empresa)

      7. telephone2 (telefone de contato empresa);

    3. Dados transacionais dos usuário solicitantes (.csv)

      1. cnpj (CNpj da empresa)

      2. period (O período que as transações deverão ser enviadas é diário - valor consolidado/dia)

      3. quantity (Quantidade de vendas dentro período enviado)

      4. amount (Total de vendas dentro do período enviado);

Para garantir a geração de uma proposta final é indispensável que  os dados apresentados sejam de no mínimo 06 meses contendo os valores diários das transações.

Obs.: Certificar-se de que a política de uso de dados da Iugu permite o compartilhamento.

  1. a55 gera as propostas;

  2. a55 cria o cadastros das empresas;

  3. a55 disponibiliza um arquivo com as propostas e um link de acesso que levará o usuário para o cadastro de senha e em seguida será inserido na etapa de aceitar a propostas e formalizar o crédito;

  4. Iugu envia por e-mail as propostas para os clientes;

  5. Cliente clica no link e é direcionado para cadastrar uma senha e acessar o aceite de proposta na plataforma da a55;

  6. Ao clicar no link o acesso do usuário é criado automaticamente, o usuário cria uma senha e acessa a etapa de aceite da proposta;

  7.  O cliente segue a jornada para finalizar a tomada de crédito (enviar documentos, assinar contrato, desembolso).
