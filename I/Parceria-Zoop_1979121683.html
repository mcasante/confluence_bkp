Created by Tamila Hohlenverger, last modified on Jun 24, 2022

Fluxo MVP Zoop - quando o parceiro só tem a LP:

Recebimento do dado:

Cliente acessa Landing pages e solicita proposta de crédito. Precisamos verificar diariamente no hubspot se tem novas solicitações, se houver, pedimos para o time da Zoop enviar a planilha de transações pelo drive (Monedd/ Mazord/Zero1pay/ Easypay/ Onepay/ Prospera/ Conbreonline/ ÉSeguroPay)

Precificação: 

Envio de base de dados via planilha no drive compartilhado com a Zoop. Time a55 roda planilha no midgard na seção de Parcerias>Geração de propostas. Caso exista um deal criado no hubspot, será necessário informar o Order ID na tela de upload da planilha.

Onde pegar Order ID: Acesse metabase > Partnership DB/public/order_receivable e filtre pelo CNPJ para idendificar o ID. https://metabase.a55.tech/question#eyJkYXRhc2V0X3F1ZXJ5Ijp7ImRhdGFiYXNlIjo1NSwicXVlcnkiOnsic291cmNlLXRhYmxlIjozMDUwfSwidHlwZSI6InF1ZXJ5In0sImRpc3BsYXkiOiJ0YWJsZSIsInZpc3VhbGl6YXRpb25fc2V0dGluZ3MiOnt9fQ==

Retorno parceiro:

Após geração da proposta, iremos disparar o e-mail com devolutiva para o cliente, e colocar no acompanhamento da Zoop (https://docs.google.com/spreadsheets/d/13j-UCScu-IDxiBty_l4u5KslBE9sKXMvMsGTyLorPEs/edit#gid=0 )

Aceite de proposta:

Cliente final dará aceite via nossa plataforma. Através da LP criada, o cliente acessará para visualizar a proposta, dar aceite e seguir o processo de formalização. 

Definir comunicação enviada alertando que a proposta está disponível (definir se será por e-mail / whatsapp). 

Formalização:

Parceiro não tem documentação, cliente final faria via nossa plataforma

Cobrança:

Cliente final pagará boleto

Enviaremos boleto por e-mail para cliente final + acesso via plataforma

Recebimento do dado:

Dado é enviado direto da integração com a plataforma Zoop. Clientes acessam banner na plataforma do parceiro e solicitam proposta.

Precificação: 

Dados enviados via API são disponibilizados para pricing rodar a política de precificação e gerar uma proposta no midgard. Após validação da mesa de crédito, a API retorna para parceiro e envia e-mail para cliente.

Retorno parceiro:

Nesse momento a Zoop está trabalhando para fazer a busca no nosso endpoint de retorno de proposta. Guilherme Ventura dos Santos (Unlicensed) pode atualizar aqui com os próximos passos de integração.

Aceite de proposta:

Cliente final é direcionado para plataforma a55 afim de dar o aceite. Através do banner de solicitação, o cliente cria um usuário para acessar a plataforma, visualizar a proposta, dar aceite e seguir o processo de formalização. 

Definir comunicação enviada alertando que a proposta está disponível (definir se será por e-mail / whatsapp). 

Formalização:

Parceiro não tem documentação, cliente final faz todo o processo via plataforma a55.

Cobrança:

Processo de cobrança é feito por boleto: Cliente final pagará boleto.

Enviaremos boleto por e-mail para cliente final + acesso via plataforma
