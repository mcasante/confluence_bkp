Descrição

A PagoLivre (https://www.pagolivre.com.br/) é uma subadquirente que lida c/ tecnologia de recorrência de pagamento focado em franquias. Hoje eles detém uma base grande de franquiados (sem números) e uma operação forte de MGM.

  1. Visão global de recebíveis

    1. Desembolso de 16MM no recebível futuro

    2. Antecipa 50% do à performar

    3. Ele já controla a trava do recebível do cliente

    4. Ele tem todos os documentos para fazer formalização

    5. Volume esperado de originação esperado: 30 novos clientes/dia

A Pago Livre trabalha com 4 adquirentes: Bling, Cielo, Global Payments, +1.

Existem dois tipos de produtos para antecipação: crédito fumaça e performado. A Pagolivre tipicamente permite a antecipação de duas maneiras:

  1. Performado (Taxa praticada atualmente: 1.99% a.m)

    1. 100% dos recebíveis (Para quem não tem crédito fumaça contratado)

    2. 50% dos recebíveis para quem tem crédito fumaça contratado, para não comprometer a agenda do que irá receber, impedindo os pagamentos futuros

  2. Não performado (fumaça)

    1. Agenda de recebíveis dos 4 adquirentes que o parceiro trabalha, com taxas de 2,49% a 2,99%

Riscos

Riscos marcados são aqueles que já temos respostas, enquanto ainda precisamos mitigar os demais riscos

Valor

  • Os clientes da Pago Livre querem crédito?
  • Na primeira versão não teremos interação entre os clientes e interfaces do lado da a55, minimizando a chance de fidelização em função da nossa marca

Usabilidade

  • Um fluxo onde enviamos propostas manualmente pode ser ruim para a experiência do usuário

Viabilidade

  • Precisamos garantir que os dados que a Pago Livre nos enviou
  • Garantir que temos infraestrutura preparada para colocarmos esses leads dentro do HubSpot
  • Garantir que temos todos os dados para precificar uma proposta firme
  • Os dados que a Pago envia para a gente são dados anonimizados? (O usuário já aceitou um optin de LGPD?)
    • No contrato principal, o usuário aceita termos dando direitos
    • Não podemos enviar um email para todo mundo que está na base de dados atualmente, vamos operar e segue a vida

Business

  • (Legal) Para termos uma obrigação de a Pago Livre repassar o dinheiro para a gente, é necessário que o fundo seja o detentor do dinheiro.
    • Pago Livre fica responsável pela gestão do dinheiro, mas a titularidade da dívida deverá ser nossa
    • O dinheiro precisa sair direto do cliente para a gente, sem passar pela Pago Livre.
  • Novos clientes não possuem histórico com a Pago Livre, como conseguimos pedir histórico?
    • A Pago Livre hoje pede 3 meses de carência, para garantir o conhecimento do histórico enquanto a a55 precisa de 6 meses do fundo (Precisamos mexer na nossa CCB)
  • Quem é o empresa a55 que deveria ser a responsável por esse contrato?
    • a55 LTDA
  • O que esperamos do parceiro quando falamos em garantia
    • 3952 - a Pago Livre é obrigada a registrar os pagamentos na registradora
  • Não podemos tercerizar a nossa responsabilidade
    • Deixar claros os documentos que precisamos ter do lado deles
    • Fazer um pente fino no processo dele
    • Qual é o processo amostral de análise que nós queremos que eles tenham (5% dos casos)
      • Podemos ter uma taxa maior para os primeiros casos → Para os 10 primeiros casos, vamos analisar 100%
  • Queremos que ele forneça para a agenda dele
    • Fazer troca de titularidade do recebível através de registradora
  • A Pago Livre precisa disponibilizar a documentação dos clientes deles para nós até D+3~D+4
    • Nós podemos liberar o crédito a partir do momento que temos o compromisso das partes na sessão do crédito.

Solução

Montaremos uma API para receber informações de forma assíncrona sobre clientes que toparam fazer a antecipação no painel do parceiro.

Inicialmente, faremos precificação sob demanda, em lotes via excel. Apesar do esforço manual, precisamos manter o fluxo de precificação contínuo.

Começaremos a operacionalizar com empresas que tenham mais fit com o nosso crédito da linha Digital, exclusivamente, pois não conseguiríamos viabilizar uma linha de Ads dentro do apetite atual.

Milestones

  1. Criar fluxo de precificação manual, recebendo dados via planilha, precificando através do Pricing Manager e devolvendo propostas manualmente para a Pago Livre.

  2. Criar fluxo automático para envio das propostas firmes para todos os clientes que solicitaram uma proposta de crédito performado ou fumaça

  3. Criar fluxo automático para troca das informações necessárias para preenchimento da CCB junto ao bancarizador

  4. Criar usuário na plataforma com as informações que foram parcialmente enviadas

  5. Originar diretamente em lote aproveitando as janelas bancárias para envio das informações durante último espaço enviado

Diagrama de processos

  1. Poucas ações do lado do cliente tanto para solicitar quanto para ter sua operação amortizada

  2. Necessidade de compatibilizar o checklist de documentação e manter os documentos dos clientes com operação vigente sempre disponíveis.

  3. Capacidade de realizar as transferências com prioridade, assim que o recurso estiver disponível no domicílio do cliente fazer os pagamentos, se for o caso.

  4. Definir se seguimos nosso processo padrão de coleta de documentos para os casos que o parceiro não tenha a documentação completa necessária para a operação acontecer.

  5. Negociar com fundo e custódia um prazo estimado para o levantamento da documentação, pós desembolso, via comprometimento jurídico das partes de apresentação dos mesmos.
