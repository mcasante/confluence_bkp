Problema

Precisamos de uma forma de coletar os dados dos clientes de nossos parceiros de forma rápida e escalável, para geração de propostas firmes, sem depender de uploads manuais, nem de direcionar esses clientes para nossa plataforma (condição do parceiro)

Solução / Hipótese

LP externa a plataforma, para conexão de dados do GA dos clientes finais de nossos parceiros

Proposta de valor única

Sermos a única provedora de crédito no Brasil que realiza análise baseada somente am dados de Google Analytics

Vantagem competitiva

Analise extremamente rápida (retorno em segundos), sem necessidade de o cliente realizar upload de nenhum documento, nem cadastrar contas bancárias

Segmento de clientes

MarketPlaces e Agencias de Marketing Digital

Riscos

Seremos responsáveis pelo armazenamento dos dados dos clientes dos nossos parceiros. Tendo em vista a LGPD, precisamos garantir que o parceiro e os seus clientes estão cientes de quais dados serão armazenados, como e por quanto tempo.

Métricas

Volume de conexões de google analytics realizadas

Volume de propostas geradas com base em dados do GA oriundas de parcerias

Volume de desembolsos de empresas analisadas via dados de GA, oriundas de parcerias

Fontes de receita

TAC dos desembolsos realizados no contexto de parcerias (cuja analise foi feita via GA)

Estimativa de Custo

Designer UX alocado por 1 sprint para prototipação da solução

Front-end dedicado 1 semana para desenvolvimento da LP

Que problema(s) estamos resolvendo?

O problema a ser resolvido é que hoje temos parceiros dispostos a nos indicar centedas de clientes para iniciarem o processo de tomada de crédito, mas esses parceiros colocaram a condição de que só direcionarão os seus clientes para nossa plataforma após eles terem recebido propostas firmes de crédito (para impedir que os clientes deles se frustrem com eventuais negativas).

Para isso, precisamos que esses parceiros tenham uma forma de nos passar os dados de seus clientes finais, para que possamos rodas nosso modelo de princing e gerar essas propostas.

Nós já temos uma API de precificação que se baseia em dados do google analytics para gerar propostas em tempo real, e ela poderá ser utilizada nesse contexto. Contudo, não temos hoje uma página na qual nosso parceiro consegue fornecer acesso as contas de seus clientes do analytics, impedindo assim a utlização dessa API e, consequentemente, o andamento dessas parcerias.

Por quê agora?

Temos um volume significativo de parceiros interessados utilizar nossas soluções de crédito para oferecer aos seus clientes finais e só estão aguardando uma forma de repassar os dados de seus clientes de forma rápida (escalável), para que possamos realizar as analises de crédito para gerar propostas firmes.

Dentre esses parceiros aguardando uma forma de nos repassar dados, vale destacar a Ame Digital (https://www.amedigital.com/), Loja Mestre (https://www.lojamestre.com.br/), Plugg.TO (https://plugg.to/), dentre outros.

Objetivos estratégicos

KPIs relacionados a iniciativa:

-50% % de leads originados através de parcerias


-Atingir 25% dos desembolsos realizados para ads e digital totalmente tech-touch (16 desembolsos)


-Atingir 100MM em desembolsos para SaaS e Ecommerce


O que define o sucesso dessa iniciativa?

Pelo menos um cliente desembolsado, proveniente de uma parceria, cuja analise de crédito tenha sido feita somente com base nos dados do GA, fornecidos não pelo cliente final, mas pela empresa parceira

Como mensuramos os resultados

Taxa de sucesso dos parceiros ao tentarem realizar o optin-in das contas de seus clientes do GA

Volume de conexões de google analytics realizadas

Volume de propostas de crédito geradas com base em dados do GA oriundas de parcerias

Volume de desembolsos de empresas analisadas via dados de GA, oriundas de parcerias

Dependências

Já temos os end-points para conexão do analytics prontos, além do modelo de pricer para a geração das propopostas firmes.

Pendente: Quais termos parceiro precisa aceitar antes de dar opti-in das contas do GA, para que estejamos cobertos quanto a LGPD.
