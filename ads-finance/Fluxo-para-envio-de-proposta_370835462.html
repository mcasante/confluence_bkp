Adsfinance : Fluxo para envio de proposta

Created by Corintho Fernandes, last modified by Douglas Wille on Oct 28, 2020

Cadastro de Propostas via Midgard

Para realizar o cadastro da proposta via Midgard (ambiente de backoffice), existem alguns pré-requisitos:

Cliente precisa estar cadastrado na plataforma;

O usuário que for criar a proposta precisa de acesso ao ambiente do Midgard como Staff;

Ambiente do Midgard: http://midgard.a55.tech/

Passo a passo:

  1. Acessar o ambiente do Midgard;

  2. Acessar no menu do ambiente Propostas (crédito);

  3. Dentro do contexto de Propostas (crédito) acessar Criar nova proposta;

  4. Selecione a empresa, o status (se foi aprovado ou recusada a linha de crédito do cliente);

  5. Preencha os campos com as taxas e adicione uma nova opção caso necessário;

    1. Cada opção nova corresponde a uma opção de proposta que será exibida para o cliente;

  6. Assim que finalizar o processo clique em Criar;

Atenção: Não é possível criar uma proposta, com alguma em aberto. Nesse caso, você deve remover as propostas atuais e criar novas.

Acompanhamento de Propostas via Midgard

Para realizar o acompanhamento e/ou gerenciamento da proposta via Midgard (ambiente de backoffice), existem alguns pré-requisitos:

Cliente precisa estar cadastrado na plataforma;

Cliente precisa ter propostas disponíveis (para que seja possível visualizar/gerenciar);

O usuário que for criar a proposta precisa de acesso ao ambiente do Midgard como Staff;

Ambiente do Midgard: http://midgard.a55.tech/

Passo a passo:

  1. Acessar o ambiente do Midgard;

  2. Acessar no menu do ambiente Propostas (crédito);

  3. Dentro do contexto de Propostas (crédito) acessar Propostas de crédito;

  4. Selecione a empresa;

  5. Veja se a empresa selecionada possui propostas em aberto e caso necessário visualize e/ou remova as propostas em aberto.

Atenção: Não é possível editar uma proposta. Você precisa remover todas as opções e criar novamente.

Antes do fluxo digital via Midgard, fazíamos o envio de propostas via Postman.

O processo manual antigo está relacionado abaixo, mantivemos apenas caso surja a necessidade de rodar manual por algum problema, mas o ideal é utilizar apenas o fluxo digital.

Fluxo manual via Postman

Este texto tem como objetivo documentar os passos para o envio de proposta de crédito, após o cadastro para análise de crédito no sistema AdFinance (https://adfinance-register.a55.tech/)

Instruções Prévias (Cadastro por parte do cliente)

Overview rápido para entender o fluxo anterior ao envio da proposta

  1. O cliente irá acessar a página de registro do AdFinance (https://adfinance-register.a55.tech/) e preencher todos os dados solicitados:
    - Nome / Sobrenome
    - Email Empresarial
    - Número do Celular
    - Nome da Empresa
    - Cargo
    - CNPJ
    - Site da empresa
    - Receita bruta mensal
    - Ferramenta de marketing usada pela empresa (Google Ads ou Facebook Ads)
    - Será necessário informar o número da conta de anúncio e dar as permissões
    - Dados financeiros (Wirecard, PagSeguro ou Conta Bancária)
    - No caso de Wirecard e PagSeguro será necessário dar permissão na plataforma

Instruções Para envio da proposta (Colaboradores A55 / AdFinance)

Entendido como o cliente enviou seus dados, agora poderemos analisar estes dados e aprovar uma ou mais propostas para ele, ou ainda rejeitar esta proposta

  1. Equipe de Pricing deverá ver os dados do cliente dentro do Metabase (https://metabase.a55.tech/)

  2. Dentro do Metabase os dados de campanha do cliente (Google Ads ou Facebook Ads) vão aparecer e poderão ser analisados para que uma proposta seja criada com base em seu desempenho:

  3. Sabendo quais propostas podem ser aprovadas para o cliente a equipe de pricing deverá alimentar a seguinte planilha com as propostas: (Atenção: Pricing é quem alimenta a planilha)
    https://docs.google.com/spreadsheets/d/1fUkbz-rBxypHuI7B3FbDmDWn2txrHWILCzHZGNCohyQ/edit#gid=0

     

  4. A planilha é acessada todos os dias por um membro da equipe do AdFinance para saber se existem propostas liberadas pela equipe de pricing (Atenção: Membro AdFinance é quem lê a planilha e atualiza se a proposta foi ou não enviada)

  5. Se uma ou mais propostas foram liberadas pela equipe de pricing, é hora de enviar (disponibilizar) a proposta dentro da plataforma da a55. Para isso faremos uso do programa Postman, que pode ser baixado em: https://www.postman.com/downloads/

  6. Ao entrar no programa Postman, o usuário deverá criar uma nova requisição, acessando o menu:
    - File → New Tab

  7. Você verá uma aba (parecida com as que vemos nos browsers/navegadores), selecione o tipo de request como “POST”

  8. Em “Enter Request URL” você deverá colocar:
    - https://proposals-api-staging.a55.tech/proposals (staging / para testes)
    - https://proposals-api.a55.tech/proposals (produção / para enviar propostas reais / ATENÇÃO)

Os campos: cnpj, email, name e status são obrigatórios para o adFinance, eles garantem que o email transacional de proposta disponível será enviado (este email será recebido pelo cliente, assim que a proposta for enviada pelo Postman)

Acompanhamento do status dentro da plataforma (Cliente)

Quando o cliente completa seu cadastro no AdFinance, mesmo que ele ainda não tenha uma proposta disponível, ele poderá acompanhar o andamento de sua análise. Acessando a plataforma:

  • https://plataforma-staging.a55.tech/ (Staging / Testes)

Dentro da plataforma ele poderá acessar “Serviços Financeiros” → “Crédito para Marketing”, e caso ele ainda não tenha passado pela análise de crédito ele verá a tela de status e estará na fase 1 “Análise de Crédito”

Enviando uma resposta negativa de crédito (Proposal Canceled)
(Colaboradores a55 / AdFinance)

Para rejeitar um pedido de crédito junto a a55, você deverá seguir o mesmo procedimento dentro do Postman que foi usado para enviar uma proposta. A diferença está no corpo do JSON (Body) da requisição que será feita. Você deverá enviar uma proposta com todos os parâmetros “zerados” e com o status “canceled”, conforme o modelo abaixo:

{
	"cnpj": 80403344000180,
    "email": "corintho@a55.tech",
    "name": "Corintho TesteLocal",
	"status": "canceled",
	"rates": [
		{
			"tac": 0,
			"grace": 0,
			"rate": 0,
			"prepaymentFee": 0,
			"amount": 0,
			"numberInstallments": 0
		}
	]
}
